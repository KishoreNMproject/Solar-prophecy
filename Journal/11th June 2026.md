### Open Investigation: Android 17 Runtime Crash
The previous Android 17 investigation is still open.
The APK installs on the Android 17 emulator, therefore the minSdk/API 17 explanation is incorrect.
Android 17 was referring to a modern Android 17 emulator image, not Android API Level 17 (Android 4.2).
Current evidence:
- APK installs successfully.
- Application icon appears in launcher.
- Problem occurs after installation/runtime.
- Therefore this is not a package installation rejection caused by minSdk.

06-11 19:08:14.051  4578  4578 E AndroidRuntime: 	at android.os.Parcel.createExceptionOrNull(Parcel.java:3392)

06-11 19:08:14.051  4578  4578 E AndroidRuntime: 	at android.content.ContextWrapper.registerReceiver(ContextWrapper.java:793)
06-11 19:08:14.051  4578  4578 E AndroidRuntime: 	at app.solarprophecy.MainActivity.onCreate(MainActivity.java:101)

#### Root Cause
On Android 14 (API 34) and above, it is strictly required to explicitly specify `RECEIVER_EXPORTED` or `RECEIVER_NOT_EXPORTED` when dynamically registering a `BroadcastReceiver`. `MainActivity.java` at line 101 was failing to specify this flag for the `DownloadManager.ACTION_DOWNLOAD_COMPLETE` intent filter, leading to an immediate startup crash.

#### Fix
Added conditional logic in `MainActivity.java` to apply `Context.RECEIVER_EXPORTED` on devices running TIRAMISU (API 33) and above:
```java
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
            registerReceiver(onDownloadComplete, new IntentFilter(DownloadManager.ACTION_DOWNLOAD_COMPLETE), Context.RECEIVER_EXPORTED);
        } else {
            registerReceiver(onDownloadComplete, new IntentFilter(DownloadManager.ACTION_DOWNLOAD_COMPLETE));
        }
```

#### Validation after fix
Compiled and installed v1.5.11 on the Android emulator (`Medium_Phone`). The `java.lang.SecurityException` no longer occurs in the logcat, and the `AndroidRuntime` fatal crash is completely resolved. The application launches successfully without immediate failure.
